Slate Systems — Role Definitions

Precise definitions for all roles. Absorbed by founders until hire trigger.
Last updated: Feb 6, 2026

Role 1: Founder

Type: Human Absorb Until: Never (always founder) Current Owner: Dan

Duties

  1. Set pricing and packaging decisions
  2. Hire/fire team members
  3. Approve refunds over $100
  4. Handle legal/compliance issues
  5. Set strategic direction
  6. Manage cash flow and finances
  7. Handle partnership/affiliate deals
  8. Final escalation point for unresolved issues
  9. Approve marketing spend over $500
  10. Negotiate vendor contracts

Triggers

  • Escalation from any role that exceeds their authority
  • Hiring need identified
  • Pricing change requested
  • Legal/compliance question
  • Strategic decision needed

Inputs

  • Escalation requests with context
  • Financial reports
  • Performance metrics from all roles
  • Market feedback

Outputs

  • Decisions (documented)
  • Hired team members
  • Strategic plans
  • Approved budgets

Tools

  • Bank accounts
  • Stripe dashboard
  • GHL agency admin
  • Hiring platforms (OnlineJobs.ph, etc.)

Handoffs

FromTo FounderTrigger
CloserPricing exception requestClient wants custom deal
CloserRefund request >$100Client demands refund
FulfillmentUnresolvable issueA2P permanently rejected, etc.
AnyLegal questionCompliance, contracts

KPIs

  • MRR growth
  • Profit margin
  • Team retention
  • Client churn rate

Decision Authority

  • ALL decisions not delegated to other roles
  • Can override any other role's decision

Escalation

  • N/A (final escalation point)

Role 2: Closer

Type: Human Absorb Until: $25K MRR Current Owner: Dan

Duties

  1. Run demo calls (30 min)
  2. Handle objections and close deals
  3. Run launch calls (30-45 min)
  4. Collect payment
  5. Request testimonials on launch call
  6. Request referrals
  7. Follow up with pending deals (not booked)
  8. Re-engage lost deals (manual, high-value)
  9. Handle cancellation save attempts

Triggers

TriggerAction
Demo bookedPrep + run demo
Demo completed, not closedFollow up sequence
Deal wonHand off to Fulfillment
Launch call bookedPrep + run launch call
Cancellation requestSave attempt call

Inputs

  • Booked demo (from GHL calendar)
  • Lead info (from GHL contact)
  • Qualification notes (from Sales Assistant agent)
  • QA-ready client (from Fulfillment)
  • Cancellation request (from client)

Outputs

  • Won deal (payment taken)
  • Lost deal (reason documented)
  • Launched client (moved to Active)
  • Saved client (cancellation prevented)
  • Testimonial collected
  • Referral collected

Tools

  • GHL (contacts, opportunities, calendar)
  • Google Meet
  • Stripe (payment links)
  • Loom (for async follow-ups if needed)

Role 3: Fulfillment Specialist

Type: Human Absorb Until: $10K MRR Current Owner: Dan

Duties

  1. Monitor new won deals daily
  2. Buy toll-free number for client
  3. Submit A2P verification
  4. Handle A2P rejections
  5. Request GMB access from client
  6. Connect GMB to GHL
  7. Populate custom values in client sub-account
  8. Configure phone forwarding
  9. QA website before launch call
  10. Fix QA issues
  11. Move client through pipeline stages
  12. Handle GMB issues (verification, recovery)
  13. Coordinate with Content Generator agent output

Triggers

Form submitted (Form Received)Start build process

GMB issue reportedTroubleshoot/escalate

Inputs

  • Won deal notification
  • Completed onboarding form (contact custom fields)
  • Content Generator agent output (generated content)
  • A2P status updates
  • GMB access grant from client
  • QA checklist

Outputs

  • Toll-free number purchased
  • A2P submitted/approved
  • GMB connected
  • Custom values populated
  • Website QA'd and ready
  • Client in "Launch Scheduled" stage

Tools

  • GHL (sub-accounts, phone numbers, custom values)
  • Google Business Profile Manager
  • imgBB (for A2P screenshots)
  • Slack/communication tool (for client questions)

Role 5: Support Agent

Type: AI Agent (OpenClaw) Absorb Until: $25K MRR (then human backup) GHL Integration: openclaw-support

Duties

  1. Monitor incoming client messages
  2. Answer routine questions
  3. Provide how-to guidance
  4. Tag contacts needing human help
  5. Notify team of escalations

Triggers

TriggerAction
Incoming message from active clientEvaluate + respond or escalate

Inputs

  • Incoming message content
  • Contact info (name, business)
  • Contact tags (currentclient, etc.)
  • Message history (if available via scope)

Outputs

  • Response message (via SMS)
  • Tag added (needs-human) if escalating
  • Notification to team (via n8n)

Tools

  • GHL API (conversations/message.write, contacts.write)
  • Claude model (via OpenClaw)
  • Knowledge base (FAQ, how-tos)

Knowledge Base (Agent Must Know)

  1. How to check messages in Lead Connector app
  2. How to use marketing form for reviews
  3. What the review system does
  4. Common scam warnings
  5. Who to contact for billing (Founder)
  6. How to reset app password
  7. A2P wait times
  8. GMB verification process

Handoffs

FromTo AgentDeliverable
ClientAgentIncoming message
From AgentToDeliverable
AnsweredClientResponse sent
EscalatedFounder/CloserTagged contact + notification

KPIs

MetricTarget
Auto-resolution rate>70%
Response time<5 minutes
Escalation rate<30%
Client satisfactionNo complaints

Decision Authority

Can DecideCannot Decide (Escalate)
Answer how-to questionsBilling questions
Explain featuresRefund requests
Reset guidanceTechnical bugs
Standard troubleshootingCustom requests

Escalation

  • Tag needs-human + notify via n8n
  • To Founder: Billing, refunds, complaints

Role 6: Sales Assistant Agent

Type: AI Agent (OpenClaw) Absorb Until: $25K MRR (then human SDR backup) GHL Integration: openclaw-sales

Duties

  1. Respond to inbound lead inquiries
  2. Qualify leads via conversation
  3. Answer pre-sale questions
  4. Send calendar link when ready
  5. Update contact fields with qualification data
  6. Tag leads appropriately

Triggers

TriggerAction
Lead replies to sequenceEngage + qualify
Lead in Follow-Up stageEngage + qualify
Lead asks questionAnswer + qualify

Inputs

  • Incoming message from lead
  • Contact info (name, business, source)
  • Contact tags
  • Lead source (FB ad, referral, etc.)

Outputs

  • Response messages (via SMS)
  • Updated contact fields:
  • Qualification notes
  • Business size
  • Timeline
  • Budget indication
  • Tags added:
  • qualified (ready for demo)
  • hot-lead (high intent)
  • not-qualified (doesn't fit)
  • Calendar link sent (when qualified)

Tools

  • GHL API (contacts.write, conversations/message.write)
  • Claude model (via OpenClaw)
  • Calendar link (custom value)
